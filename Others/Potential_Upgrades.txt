plugin idea:
Your teacher is basically asking you to evolve Optivox from a **normal application → a background accessibility service**.
Windows accessibility tools work exactly this way. The program sits in memory and wakes up only when needed.

Think of it as turning Optivox into a **resident helper**, not a program you open manually.

Let’s break the requirements into engineering pieces.

---

First requirement: **always active plugin**

That just means the program runs in the background and stays alive.

You already almost have this with:

* system tray icon
* background threads
* magnifier modules

Instead of launching magnifier modules from the GUI, you run a **background controller** that listens for keyboard shortcuts.

Example idea:

Program runs → hidden → waits for hotkeys → launches modules.

Python libraries that do this well:

```
keyboard
pynput
```

Example hotkey listener:

```python
import keyboard

def open_interface():
    print("Opening Optivox UI")

keyboard.add_hotkey("ctrl+alt+o", open_interface)

keyboard.wait()
```

Now Optivox behaves like a **system tool**.

---

Second requirement: **box magnifier around cursor**

Right now you display a separate floating window.

Your teacher wants a **lens magnifier**, like Windows Magnifier's lens mode.

Concept:

Instead of placing a window near the cursor, you center the magnifier **on the cursor itself**.

Steps:

1. Capture region around mouse
2. Scale image
3. Draw border (optional)
4. Display centered on cursor

So change positioning from:

```
self.move(mx + offset_x, my + offset_y)
```

to:

```
self.move(mx - target_w//2, my - target_h//2)
```

Now the magnifier becomes a **cursor box lens**.

Add a border:

```python
cv2.rectangle(magnified, (0,0), (target_w-1,target_h-1), (0,255,0), 2)
```

This visually shows the magnified region.

---

Third requirement: **auto start with Windows**

This is classic Windows behavior.

Two common methods.

Simplest method (recommended):

Add Optivox to the **Startup folder**.

Path:

```
Win + R
shell:startup
```

Place a shortcut to:

```
python gui.py
```

Now it launches every time Windows starts.

---

Better professional method:

Use **Windows registry startup entry**.

Example Python installer code:

```python
import winreg

key = winreg.OpenKey(
    winreg.HKEY_CURRENT_USER,
    r"Software\Microsoft\Windows\CurrentVersion\Run",
    0,
    winreg.KEY_SET_VALUE
)

winreg.SetValueEx(key, "Optivox", 0, winreg.REG_SZ,
                  r"C:\Path\To\optivox.exe")
```

That’s how real accessibility tools register themselves.

---

Fourth requirement: **keyboard shortcut to open interface**

Just add a global hotkey.

Example:

```
Win + O
```

Implementation:

```python
keyboard.add_hotkey("windows+o", open_interface)
```

Or mimic Windows Magnifier:

```
Win + =
```

Example:

```python
keyboard.add_hotkey("windows+=", start_magnifier)
keyboard.add_hotkey("windows+-", stop_magnifier)
```

The architecture of Optivox now becomes something like this:

```
Optivox Background Service
        │
        │
        ├── Hotkey Listener
        │       ├── Win + O → open UI
        │       ├── Win + = → magnifier
        │       └── Win + R → reader
        │
        ├── Cursor Lens Magnifier
        │
        ├── Screen Readers
        │
        └── Voice Assistant
```

The GUI becomes **optional control panel**, not the main program.

---

One warning worth knowing.

Libraries like `keyboard` require **administrator privileges** for global hotkeys sometimes.
`pynput` avoids that problem.

Example alternative:

```
pip install pynput
```

1. Computer starts

4. Press hotkey → interface opens

6. Press another hotkey → voice assistant activates (far future)

Exactly like built-in accessibility tools.
